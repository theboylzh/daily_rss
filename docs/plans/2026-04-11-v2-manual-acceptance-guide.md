## 第二步：AI 提示词调优（迭代）

### 2.1 定位提示词文件

提示词位于 `ai_analyzer_v2.py` 中：

```python
def _get_stage1_prompt(self) -> str:  # 阶段 1：概要生成
def _get_stage2_prompt(self) -> str:  # 阶段 2：观点生成
def _get_stage3_prompt(self) -> str:  # 阶段 3：深度分析
def _get_stage4_prompt(self) -> str:  # 阶段 4：建议生成
```

### 2.2 常见问题与调优策略

#### 问题 1：JSON 格式不稳定

**现象**：AI 输出包含 Markdown、额外说明文字

**调优方法**：在提示词开头强化要求：

```python
"""
任务=分析以下新闻列表，生成结构化的 JSON 输出。

【重要要求】
1. 必须输出纯 JSON，不要任何 Markdown 格式
2. 不要任何```json```包裹
3. 不要任何额外说明文字
4. JSON 必须能被直接解析

输出结构=...
"""
```

#### 问题 2：one_liner 过长

**现象**：输出超过 30 字

**调优方法**：

```python
one_liner: 一句话总结（严格限制 20-30 字，超过会出错）
示例："AI 行业震动：Sora 关停，巨头加速布局，油价影响消费"
```

#### 问题 3：观点空洞

**现象**：perspectives 的 description 太笼统

**调优方法**：在提示词中添加论述框架：

```python
description 论述要求：
1. 先描述现象/事实
2. 分析背后的原因/趋势
3. 指出影响/启示
4. 200-300 字
```

#### 问题 4：分析字段混淆

**现象**：facts 和 viewpoint 分不清

**调优方法**：明确字段定义：

```python
facts: 客观事实（谁、何时、何地、做了什么）
  - 必须有明确的主体
  - 必须有具体的时间/场合
  - 不能包含主观评价
  
viewpoint: 整体观点（这件事的意义/影响/好坏）
  - 表达看法和判断
  - 说明影响级别
  - 可以是正面/负面/中性
```

### 2.3 调优流程

```
1. 运行分析 → python3 ai_analyzer_v2.py
2. 查看输出 → cat data/analysis/YYYY-MM-DD_v2.json
3. 发现问题 → 记录具体问题（如"one_liner 太长"）
4. 修改提示词 → 编辑 ai_analyzer_v2.py
5. 重新运行 → 验证改进效果
6. 重复 1-5 直到满意
```

## 常见问题 FAQ

### Q1: JSON 解析失败率高

**A**: 检查 AI 输出是否包含 Markdown，在提示词中强化"纯 JSON 输出"要求，或增加解析器的 Markdown 提取能力。

### Q2: 分析内容太笼统

**A**: 在提示词中添加具体要求和示例，如"facts 必须包含时间、主体、具体事件"。

### Q3: 邮件样式错乱

**A**: 检查是否使用 table 布局，避免使用复杂 CSS，确保 CSS 内联。

### Q4: Token 消耗过高

**A**: 缩短 Prompt 长度，限制 AI 输出长度（max_tokens），分批处理新闻。
